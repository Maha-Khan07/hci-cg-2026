Introduction to GOMS

The GOMS model (Card, Moran, and Newell) is a cognitive model used to analyze human-computer interaction. GOMS stands for:

  • Goals: The user’s objectives, which act as memory points for evaluating actions.
  • Operators: Basic actions performed by the user, such as pressing a key or reading a dialog.
  • Methods: Sequences of operators to achieve a goal; multiple methods may exist for the same goal.
  • Selection rules: Rules that determine which method is chosen depending on context or user behavior.

Components Explained

Goals

Goals describe what the user wants to achieve. They serve as reference points, allowing evaluation and correction if errors occur.

Operators

Operators are the smallest unit of action in the model. They can be physical (e.g., press 'X') or mental (e.g., read a dialog box). Granularity depends on analysis.

Methods

Methods are ways to achieve a goal by decomposing it into subgoals. For example, closing a window can be done by:

  • CLOSE-METHOD: Using a pop-up menu
  • L7-METHOD: Pressing a function key

Selection Rules

Selection rules determine which method to use.They depend on the particular user ,the system and the system state Example:

  • User Sam: Rule 1 – Use CLOSE-METHOD unless another rule applies.
  • User Sam: Rule 2 – If application is 'blocks', use L7-METHOD.

Example: Editing a Manuscript

High-level goals and subgoals:

  • GOAL: EDIT-MANUSCRIPT
    • GOAL: EDIT-UNIT-TASK (repeat until all unit tasks are done)
    • Operators: Basic actions like typing, deleting, moving cursor

Example: Photocopying an Article

  • GOAL: PHOTOCOPY-PAPER
    • GOAL: LOCATE-ARTICLE
    • GOAL: PHOTOCOPY-PAGE
      • GOAL: ORIENT-PAGE
      • OPEN-COVER
      • SELECT-PAGE
      • POSITION-PAGE
      • CLOSE-COVER
      • GOAL: PRESS-COPY-BUTTON
      • GOAL: VERIFY-COPY
    • GOAL: COLLECT-COPY
      • LOCATE-OUT-TRAY
      • EXAMINE-COPY
      • REMOVE-COPY
    • GOAL: RETRIEVE-JOURNAL

The closure problem occurs if the original journal is not removed from the copier while the copy has been collected. GOMS helps identify these issues.

Comparison with Norman's Seven Stages of Action

  • Norman emphasizes user perception, intention, and evaluation (cognition-focused).
  • GOMS emphasizes detailed tasks and operators to predict performance (action-focused).
  • Norman explains why users act; GOMS explains what they do and how efficiently.

Design Focus

GOMS has practical applications in design. For example, NYNEX used GOMS to analyze operator performance before installing a new system, identifying that it would slow down call processing, saving millions of dollars by avoiding implementation.
