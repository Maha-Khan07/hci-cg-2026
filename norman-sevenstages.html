Overview of the Interaction Model

The interaction cycle consists of two main phases: execution and evaluation. The user first plans and executes actions, then observes and evaluates the system’s response.

The Seven Stages of Action

  1. Forming a goal
  2. Forming an intention
  3. Specifying an the action sequence
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome

Gulfs of Execution and Evaluation

The gulf of execution is the gap between what the user wants to do and what the system allows.

The gulf of evaluation is the gap between the system’s output and the user’s ability to understand whether the goal was achieved.

Analysis of the “Who Paid?” Problem (Splitwise) Using Norman’s Action Cycle
Norman’s Stage User Thought / Action Social & Emotional Impact
Goal The user wants to know who has paid and who still owes money so the group stays fair. Avoids awkward conversations and reduces tension between friends.
Plan The user plans to check the Splitwise app instead of asking people directly. Helps the user feel more comfortable and less confrontational.
Specify The user decides to open the app, select the group or trip, and view expenses. Clear steps reduce stress and confusion during group situations.
Execute The user taps through the app and opens the payment details. Smooth interaction prevents frustration and impatience.
Perceive The user sees names, amounts paid, and balances on the screen. Clear information builds trust among group members.
Interpret The user understands who has paid and who has not. Prevents misunderstandings and false assumptions.
Compare The user checks if the information matches the goal of fairness and shared responsibility. Helps maintain healthy relationships and group harmony.
