Model-based evaluation: what each model explains vs. predicts
Model Category What it Explains What it Predicts
Norman Cognitive / descriptive Understanding, errors, feedback, gulfs Nothing (no performance prediction)
GOMS Predictive / analytic Task structure for expert users Relative efficiency
KLM Predictive / quantitative Low-level physical and motor actions Execution time

KLM Operators

The Keystroke-Level Model (KLM) decomposes the execution phase of interaction into primitive operators. KLM assumes the user is expert and the method has already been chosen.

KLM Operators
Operator Name Meaning Example
KKeystrokingPressing a keyTyping a character
BButton pressPressing a mouse buttonLeft-click
PPointingMoving pointer to a targetMove cursor to icon
HHomingSwitching hand between devicesKeyboard → mouse
DDrawingDrawing lines using pointing deviceDragging to draw shape
MMental preparationPreparing for actionPause before clicking control
RSystem responseWaiting for system feedbackApplication loading

Worked Example: Correcting a Single Character

Task: user notices a single-character error, deletes it, retypes it, and returns to insertion point.

  1. Move hand to mouse
  2. Position mouse after bad character
  3. Return to keyboard
  4. Delete character
  5. Type correction
  6. Reposition insertion point

KLM operator sequence:

H[mouse]
P B[LEFT]
H[keyboard]
M K[DELETE]
K[char]
H[mouse] M P B[LEFT]

This sequence models only execution-level actions. High-level planning and interpretation are not represented.

Key Takeaways

The Keystroke-Level Model consists of seven primitive operators (K, B, P, H, D, M, R) used to predict execution time for expert, error-free interaction. It is a low-level instance of GOMS.

KLM models execution only; it does not include thinking or planning.
